# Contributing

Thanks for your interest in CapNav-Lite.

## Ground rules
- Keep safety first. If you touch motion control, include a safety review note in the PR.
- Prefer readable, testable code over cleverness.
- Keep dependencies light. CPU-only is a design requirement.

## Suggested workflow
1. Create an issue describing the change and the safety impact.
2. Add unit tests or a reproducible simulation scenario.
3. Submit a PR with a short "risk and mitigation" section.
